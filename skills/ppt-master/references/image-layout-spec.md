# 图片布局规范（强制）

含图片页面的布局规则。策略师规划阶段和执行器生成阶段都必须遵循。

**核心原则：根据图片原始宽高比计算布局，确保图片完整显示，无多余留白或裁剪。**

---

## 布局决策流程

```
1. 获取图片原始尺寸 → 计算比例（宽/高）
2. 根据比例选择布局类型
3. 计算图片最大显示尺寸
4. 分配剩余空间给文字区
5. 将结果填入设计规范中的图片资源列表
```

**执行时机**：如果图片方式包含"B) 用户提供"，在策略师完成八项确认后、内容分析和大纲前，必须运行扫描并填入图片资源列表。

---

## 布局类型选择（强制）

| 图片比例 | 布局类型 | 图片位置 | 描述 |
|---------|---------|---------|------|
| > 2.0（超宽） | 上下分割 | 上全宽 | 图片横跨画布宽度，高度按比例 |
| 1.5-2.0（宽） | 上下分割 | 上 | 图片宽度=内容区宽度，高度按比例 |
| 1.2-1.5（标准） | 左右分割 | 左 | 图片高度优先适配，宽度按比例 |
| 0.8-1.2（方形） | 左右分割 | 左 | 图片高度=内容区高度，宽度按比例 |
| < 0.8（竖图） | 左右分割 | 左 | 图片高度=内容区高度，宽度按比例 |

> 边界情况：当比例处于边界（如1.5）时，根据文字量决定。文字多 → 左右；文字少 → 上下。

## 尺寸计算公式

### PPT 16:9 (1280x720) 画布参数

```
画布: 1280 x 720 px
内容区: 1160 x 640 px（左右边距60px，上下边距40px）
标题区高度: 60 px
内容起始: y = 80 px（标题 + 间距）
```

### 上下布局计算

```
图片宽度 = W = 1160 px
图片高度 = W / R = 1160 / R px
文字区高度 = H - 图片高度 - 间距(20px)

验证：文字区高度 >= 150px（至少3-4行文字）
如不满足 → 切换到左右布局
```

### 左右布局计算

**方式1（高度优先，适用于竖图）**：
```
图片高度 = H = 600 px
图片宽度 = H x R = 600 x R px
文字区宽度 = W - 图片宽度 - 间距(20px)
```

**方式2（宽度约束，适用于宽图转左右）**：
```
图片宽度 = W x 0.7 = 812 px
图片高度 = 图片宽度 / R
文字区宽度 = W - 图片宽度 - 间距(20px)
```

**验证**：文字区宽度 >= 280px；否则减小图片区宽度。

## 布局示例

### 超宽图（比例2.45）

```
原始: 1960x800, R=2.45 → 上下分割
图片: 1160x473, 文字区: 1160x147 → 7:3 上下
```

### 标准横图（比例1.38）

```
原始: 1614x1171, R=1.38 → 左右分割
图片: 773x560（左）, 文字区: 367x560（右） → 7:3 左右
```

### 宽图边界情况（比例1.75）

```
原始: 1820x1040, R=1.75
尝试上下: 图片高度=663, 文字区=-43 ❌
切换到左右: 图片780x446（左）, 文字区360x600（右） → 7:3 左右
```

## 禁止做法

| 禁止 | 正确做法 |
|------|---------|
| 固定50:50或任意比例 | 基于图片比例动态计算 |
| 将宽图强制放入方形容器 | 使用上下布局或增大图片区宽度 |
| 将竖图放入窄横条 | 使用左右布局，图片在左 |
| 图片留白超过10% | 重新计算布局或选择替代方案 |
| 裁剪关键图片内容 | 使用 `preserveAspectRatio="xMidYMid meet"` |
| 文字区太小无法阅读 | 确保文字区 >= 150px（上下）或 >= 280px（左右） |

## SVG图片嵌入代码

### 完整显示（推荐用于数据图表）

```xml
<image href="../images/xxx.png"
       x="60" y="80" width="780" height="446"
       preserveAspectRatio="xMidYMid meet"/>
```

### 裁剪填充（仅用于背景）

```xml
<image href="../images/bg.png"
       x="0" y="0" width="1280" height="720"
       preserveAspectRatio="xMidYMid slice"/>
```

## 图片资源列表模板

设计规范与内容大纲中的图片资源列表必须包含：

| 字段 | 描述 | 示例 |
|------|------|------|
| 文件名 | 图片文件名 | `p12_0.png` |
| 原始尺寸 | 宽 x 高 | 1524x968 |
| 比例 | 宽 / 高 | 1.57 |
| 页码 | 使用页面 | 第5页 |
| 类型 | 视觉类型 | Background / Photography / Illustration / Diagram / Decorative |
| 布局方案 | 上下/左右 + 分割比例 | Top-bottom 6:4 或 Left-right 7:3 |
| 图片区 | 图片显示尺寸 | 1160x420 或 780x446 |
| 文字区 | 剩余空间尺寸 | 1160x200 或 360x600 |

**类型字段供图片生成器选择适当的提示策略。**

## 自动化工具

```bash
python3 scripts/analyze_images.py <项目路径>/images
```

输出包含尺寸、比例和布局建议（Markdown表格），可直接用于填充图片资源列表。
